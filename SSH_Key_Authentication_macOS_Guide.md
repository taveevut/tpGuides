# คู่มือการตั้งค่า SSH Key Authentication บน macOS

คู่มือนี้อธิบายวิธีตั้งค่า SSH Key เพื่อเชื่อมต่อ Server โดยไม่ต้องกรอกรหัสผ่านทุกครั้ง

------------------------------------------------------------------------

## ข้อดี

-   ไม่ต้องกรอกรหัสผ่านทุกครั้ง
-   ปลอดภัยกว่าการใช้ Password
-   สามารถยกเลิกสิทธิ์ของแต่ละเครื่องได้
-   เปลี่ยนรหัสผ่าน Server ได้โดยไม่กระทบ SSH Key
-   รองรับการใช้งานหลาย Server และหลายเครื่อง

------------------------------------------------------------------------

## 1. สร้าง SSH Key บน macOS

``` bash
ls ~/.ssh
ssh-keygen -t ed25519 -C "MacBook-Pro"
```

ไฟล์ที่ได้

``` text
~/.ssh/id_ed25519
~/.ssh/id_ed25519.pub
```

-   `id_ed25519` คือ Private Key (ห้ามเผยแพร่)
-   `id_ed25519.pub` คือ Public Key

------------------------------------------------------------------------

## 2. ติดตั้ง Public Key บน Server

วิธีที่แนะนำ

``` bash
ssh-copy-id root@SERVER_IP
```

หรือคัดลอก Public Key

``` bash
cat ~/.ssh/id_ed25519.pub
```

แล้วนำไปวางใน

``` text
~/.ssh/authorized_keys
```

กำหนดสิทธิ์

``` bash
chmod 700 ~/.ssh
chmod 600 ~/.ssh/authorized_keys
```

------------------------------------------------------------------------

## 3. ทดสอบ

``` bash
ssh root@SERVER_IP
```

------------------------------------------------------------------------

## 4. ตั้งค่า SSH Config

ไฟล์

``` bash
nano ~/.ssh/config
```

ตัวอย่าง

``` text
Host sv1
    HostName 192.168.1.10
    User root
    IdentityFile ~/.ssh/id_ed25519
```

เชื่อมต่อ

``` bash
ssh sv1
```

------------------------------------------------------------------------

## 5. ใช้ macOS Keychain

``` bash
ssh-add --apple-use-keychain ~/.ssh/id_ed25519
```

เพิ่มใน `~/.ssh/config`

``` text
Host *
    AddKeysToAgent yes
    UseKeychain yes
    IdentityFile ~/.ssh/id_ed25519
```

------------------------------------------------------------------------

## 6. เปลี่ยนรหัสผ่าน Root

``` bash
passwd
```

หรือ

``` bash
passwd root
```

SSH Key ยังคงใช้งานได้ตามปกติ

------------------------------------------------------------------------

## 7. ยกเลิกเครื่องใดเครื่องหนึ่ง

แก้ไข

``` bash
nano ~/.ssh/authorized_keys
```

ลบบรรทัดของ Key ที่ไม่ต้องการ

------------------------------------------------------------------------

## 8. ยกเลิก SSH Key ทั้งหมด

``` bash
rm ~/.ssh/authorized_keys
```

------------------------------------------------------------------------

## 9. เปลี่ยน SSH Key ใหม่

``` bash
ssh-keygen -t ed25519 -C "MacBook-Pro-New"
```

เพิ่ม Public Key ใหม่ แล้วลบ Key เก่า

------------------------------------------------------------------------

## 10. หาก Mac หาย

SSH เข้า Server จากเครื่องอื่น แล้วลบ Key ของเครื่องนั้นออกจาก

``` text
~/.ssh/authorized_keys
```

------------------------------------------------------------------------

## Best Practices

-   ใช้ SSH Key แทน Password
-   ตั้งชื่อ Key ตามชื่อเครื่อง
-   ห้ามเผยแพร่ Private Key
-   สำรอง Private Key ไว้ในที่ปลอดภัย
-   ลบ Key ออกจาก Server เมื่อเครื่องสูญหาย

------------------------------------------------------------------------

## คำสั่งที่ใช้บ่อย

``` bash
ssh-keygen -t ed25519
ssh-copy-id root@SERVER_IP
ssh root@SERVER_IP
ssh sv1
passwd
cat ~/.ssh/id_ed25519.pub
cat ~/.ssh/authorized_keys
rm ~/.ssh/authorized_keys
```
